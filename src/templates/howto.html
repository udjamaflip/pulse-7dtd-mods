How to Install Mods

Installation guide, folder structure, and version compatibility reference.

New Mod
Where is the Mods Folder?

7 Days to Die loads mods from a Mods folder in the game installation directory. Create it if it doesn't exist.

PlatformDefault Path
Windows (Steam) C:\Program Files (x86)\Steam\steamapps\common\7 Days to Die\Mods\
Windows (AppData) %APPDATA%\7DaysToDie\Mods\
Linux (Steam) ~/.steam/steam/steamapps/common/7 Days to Die/Mods/
macOS ~/Library/Application Support/Steam/steamapps/common/7 Days to Die/Mods/

If neither location shows the game, right-click the game in Steam → Manage → Browse local files to find the correct path.

Installing a Mod
  1. Download the mod ZIP from the My Mods page.
  2. Open the ZIP — it contains a single folder (e.g. MyWeapon/).
  3. Copy that folder into your Mods/ directory.
  4. Verify the structure looks like: 
    Mods/
  MyWeapon/
    ModInfo.xml     ← must be here, not in a subfolder
    README.txt
    Config/
      items.xml
  5. Launch 7 Days to Die — mods are loaded automatically on startup.
  6. Open the in-game console with F1 after loading to check for XML errors.
Always back up your save files before installing new mods.

Troubleshooting
Mod not showing any effect

Check that ModInfo.xml is directly inside the mod folder (not in a subfolder). Press F1 in-game to check the console for load errors.

XML parse error in console

Open the generated XML file in a text editor and look for syntax issues. Common causes: mismatched tags, invalid attribute values, or XPath targeting a node that doesn't exist in your game version.

XPath error — node not found

The vanilla XML files in your installation may use different node names than expected. Compare the generated XPath against the vanilla file at 7 Days to Die/Data/Config/<filename>.xml.

Crash on startup

Disable mods one by one by removing their folder from Mods/ to identify the conflict. Game logs are in %APPDATA%\7DaysToDie\ on Windows.

Wrong game version targeted

Each mod's ZIP includes a README.txt with the target version and known caveats. Re-generate the mod with the correct version selected.
